# 年报/ESG报告爬取规格说明书

## 1. 项目概述

**目标**：基于巨潮资讯网（cninfo.com.cn），批量抓取指定上市公司2018–2024年度的年报、ESG报告及社会责任报告，用于后续环境披露文本分析与披露可信代理指标抽取。

**最终用途**：对报告文本进行NLP分析，构建环境披露可信度指标体系。

## 4. 公告关键词

| 优先级 | 关键词                              | 说明                          |
|--------|-------------------------------------|-------------------------------|
| P0     | `年度报告`                          | 必须，主文档                  |
| P1     | `ESG报告` / `环境、社会及管治报告`  | 重要，ESG信息披露             |
| P1     | `社会责任报告` / `可持续发展报告`   | 重要，社会责任披露            |
| P2     | `环境报告书` / `环境信息披露`       | 补充，环境专项报告            |

> 关键词之间用 "OR" 逻辑。同一公司同一财年可能匹配到多份公告，需全部下载并按类型标注。

## 5. 公告类型

| 巨潮分类代码 | 类型名称         | 说明                      |
|--------------|------------------|---------------------------|
| category_kjnd | 年度报告全文     | 主要文档，含财务报表与环境附注 |

**注意**：巨潮资讯网对年度报告统一归入「定期报告」大类，子类为「年度报告全文」；ESG/社会责任报告归入「公司治理」或「临时公告」类别，需通过关键词过滤。

## 7. 多公告匹配策略

同一公司 + 同一财年可能产出多份匹配公告（例如：年报 + ESG报告 + 社会责任报告 + 环境报告书），匹配规则如下：

1. **公司+年份**作为唯一分组键；
2. 同一组内按公告类型标记：
   - `annual_report` — 年度报告（必须）
   - `esg_report` — ESG/环境社会管治报告
   - `social_responsibility` — 社会责任/可持续发展报告
   - `environment_report` — 环境报告书
3. 下载后文件命名规则：
   ```
   {股票代码}_{年份}_{类型}.pdf
   ```
   示例：`600019_2022_annual_report.pdf`、`600019_2022_esg_report.pdf`
4. 元数据 CSV 中记录每份文件对应的公司、代码、年份、类型、发布时间、下载URL、下载时间。

## 8. 数据来源与下载入口

- **平台**：巨潮资讯网 — http://www.cninfo.com.cn
- **搜索API**：`POST http://www.cninfo.com.cn/new/hisAnnouncement/query`
  - 支持股票代码、页码、分类、关键词检索
- **公告详情/PDF下载**：`GET http://static.cninfo.com.cn/{announcement_path}`

---

## 9. 限速与反爬策略

| 措施                 | 参数/方法                          |
|----------------------|-------------------------------------|
| 请求间隔             | 每请求后随机睡眠 **3–8 秒**         |
| 重试机制             | 失败请求最多重试 **3 次**，间隔递增 |
| User‑Agent 轮换      | 从预置 UA 池中随机选取              |
| 并发控制             | **单线程**，不启用多线程/异步       |
| 日请求上限           | ≤ 3000 次/天（实际≈500–800次/天）   |
| 高峰期规避           | 避开交易日 09:30–11:30、13:00–15:00 |
| 会话保持             | 使用 `requests.Session` + 固定 Cookie |

---

## 10. 失败记录与断点续爬

- **progress.json**：记录每个股票+年份+类型的爬取状态（`pending` / `success` / `failed` / `not_found`）
- **errors.csv**：记录失败的请求信息（代码、年份、类型、错误原因、时间戳）
- **断点续爬逻辑**：启动时读取 `progress.json`，只处理 `pending` 和 `failed` 条目
- PDF 空文件检测：下载后校验文件大小，< 5KB 视为无效并标记 `failed`

## 12. 脚本功能要求

1. 从 `crawl_spec.md` 自身或内联配置读取股票池、年份、关键词；
2. 调用巨潮 `hisAnnouncement/query` API 查询公告列表；
3. 按关键词+分类过滤，下载 PDF；
4. 遵从限速策略、重试机制；
5. 实时更新 `progress.json` 和 `metadata.csv`；
6. 支持断点续爬；
7. 每次运行结束时打印摘要统计。

---

## 13. 数据合规与伦理

- 仅用于学术研究，不用于商业用途；
- 不恶意高频请求，不绕过法律法规限制；
- 下载后数据不对外公开传播；
- 遵守巨潮资讯网 `robots.txt` 及使用条款。
